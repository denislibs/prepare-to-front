# Разница между кнопкой и ссылкой в HTML?

Кнопка (`<button>`) и ссылка (`<a>`) имеют разные назначения: кнопка для действий на странице, ссылка для навигации. Правильный выбор улучшает семантику, доступность и UX.

## Разница между кнопкой и ссылкой

### **`<button>`** — Действие

Используется для выполнения действий на странице.

**Характеристики:**
- ✅ Выполняет действие (JavaScript, отправка формы)
- ✅ Остается на той же странице (обычно)
- ✅ Семантически указывает на действие
- ✅ Может содержать HTML контент

### **`<a>`** — Навигация

Используется для перехода на другую страницу или якорь.

**Характеристики:**
- ✅ Переход на другую страницу/якорь
- ✅ Изменяет URL (обычно)
- ✅ Семантически указывает на навигацию
- ✅ Может содержать только текст и строчные элементы

## Примеры использования

### Кнопка — для действий:

```html
<button type="button" onclick="handleClick()">
    Выполнить действие
</button>

<button type="submit">Отправить форму</button>

<button type="button" aria-label="Удалить">
    <svg>...</svg>
</button>
```

### Ссылка — для навигации:

```html
<a href="/page.html">Перейти на страницу</a>

<a href="#section">Перейти к секции</a>

<a href="https://example.com" target="_blank">
    Внешний сайт
</a>
```

## Когда использовать что?

### Используйте `<button>` когда:

- ✅ Нужно выполнить действие на странице
- ✅ Отправка формы
- ✅ JavaScript обработчики
- ✅ Модальные окна, выпадающие меню
- ✅ Действия, не связанные с навигацией

### Используйте `<a>` когда:

- ✅ Нужна навигация
- ✅ Переход на другую страницу
- ✅ Якорные ссылки
- ✅ Скачивание файлов
- ✅ Внешние ссылки

## Неправильное использование

### ❌ Ссылка как кнопка:

```html
<a href="#" onclick="handleAction(); return false;">
    Выполнить действие
</a>
```

**Проблемы:**
- Нет семантики действия
- Плохая доступность
- Неправильное использование

**Правильно:**
```html
<button type="button" onclick="handleAction()">
    Выполнить действие
</button>
```

### ❌ Кнопка как ссылка:

```html
<button onclick="window.location.href='/page.html'">
    Перейти
</button>
```

**Проблемы:**
- Нет семантики навигации
- Не работает с правой кнопкой мыши
- Нельзя открыть в новой вкладке

**Правильно:**
```html
<a href="/page.html">Перейти</a>
```

## Стилизация

### Кнопка, выглядящая как ссылка:

```html
<button type="button" class="link-button">
    Выглядит как ссылка
</button>
```

```css
.link-button {
    background: none;
    border: none;
    color: blue;
    text-decoration: underline;
    cursor: pointer;
}
```

### Ссылка, выглядящая как кнопка:

```html
<a href="/page.html" class="button-link">
    Выглядит как кнопка
</a>
```

```css
.button-link {
    display: inline-block;
    padding: 10px 20px;
    background: blue;
    color: white;
    text-decoration: none;
    border-radius: 5px;
}
```

## Доступность

### Правильная доступность:

```html
<!-- Кнопка -->
<button type="button" aria-label="Закрыть окно">
    ×
</button>

<!-- Ссылка -->
<a href="/page.html" aria-label="Перейти на страницу">
    Перейти
</a>
```

## Заключение

**Разница:**

- **`<button>`**: Действие на странице (JavaScript, формы)
- **`<a>`**: Навигация (переход на другую страницу)

**Использование:**
- Кнопка для действий
- Ссылка для навигации

**Рекомендации:**
- Не используйте ссылку для действий
- Не используйте кнопку для навигации
- Выбирайте элемент по назначению

Правильный выбор элемента улучшает семантику, доступность и UX.

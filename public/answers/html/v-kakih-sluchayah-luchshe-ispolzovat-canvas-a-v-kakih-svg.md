# В каких случаях лучше использовать `<canvas>`, а в каких `<svg>`?

Выбор между Canvas и SVG зависит от задачи: Canvas для производительности и сложной графики, SVG для векторной графики и доступности.

## Используйте Canvas когда:

### 1. **Игры и интерактивные приложения**

```javascript
// Высокая производительность для игр
function gameLoop() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    // Отрисовка игры
    requestAnimationFrame(gameLoop);
}
```

### 2. **Сложные анимации**

Множество движущихся объектов, частицы, эффекты.

### 3. **Обработка изображений**

Фильтры, эффекты, пиксельная обработка.

### 4. **Графики с большим количеством данных**

Тысячи точек данных, сложные визуализации.

### 5. **Пиксельная графика**

8-bit стиль, пиксельное искусство.

## Используйте SVG когда:

### 1. **Иконки и логотипы**

```html
<svg width="24" height="24">
    <path d="M12 2L2 7v10l10 5 10-5V7z"/>
</svg>
```

### 2. **Простая векторная графика**

Диаграммы, схемы, иллюстрации.

### 3. **Когда нужна доступность**

Текст доступен для скринридеров.

### 4. **Масштабирование без потери качества**

Логотипы, иконки, которые должны масштабироваться.

### 5. **Стилизация через CSS**

```css
svg path {
    fill: blue;
    transition: fill 0.3s;
}
```

## Заключение

**Canvas:**
- Игры, сложные анимации
- Высокая производительность
- Обработка изображений

**SVG:**
- Иконки, логотипы
- Векторная графика
- Доступность
- Масштабирование

Выбор зависит от требований: производительность → Canvas, качество и доступность → SVG.

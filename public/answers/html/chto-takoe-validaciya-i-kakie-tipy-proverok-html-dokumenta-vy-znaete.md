# Что такое валидация? И какие типы проверок HTML документа вы знаете?

Валидация HTML — это процесс проверки HTML-документа на соответствие стандартам и спецификациям W3C. Валидация помогает выявить ошибки в разметке, улучшить качество кода и обеспечить правильное отображение страницы во всех браузерах.

## Что такое валидация?

Валидация — это автоматическая проверка HTML-документа на соответствие правилам и стандартам, определенным в спецификации HTML. Валидатор анализирует структуру документа, синтаксис, использование элементов и атрибутов.

### Цели валидации:

- ✅ **Обнаружение ошибок** — находит синтаксические и структурные ошибки
- ✅ **Обеспечение совместимости** — гарантирует работу во всех браузерах
- ✅ **Улучшение доступности** — выявляет проблемы доступности
- ✅ **Оптимизация SEO** — помогает поисковым системам правильно индексировать страницу
- ✅ **Поддержка кода** — облегчает поддержку и отладку

## Типы проверок HTML документа

### 1. **Синтаксическая проверка (Syntax Validation)**

Проверка правильности синтаксиса HTML:

**Проверяет:**
- Правильность закрытия тегов
- Корректность атрибутов
- Правильность вложенности элементов
- Наличие обязательных атрибутов

**Примеры ошибок:**
```html
<!-- Неправильно закрыт тег -->
<p>Текст</div>

<!-- Неправильная вложенность -->
<span><div>Блок</div></span>

<!-- Отсутствует обязательный атрибут -->
<img src="photo.jpg">  <!-- Нет alt -->
```

### 2. **Структурная проверка (Structure Validation)**

Проверка структуры документа:

**Проверяет:**
- Наличие обязательных элементов (`<!DOCTYPE>`, `<html>`, `<head>`, `<body>`)
- Правильный порядок элементов
- Соответствие DOCTYPE используемым элементам
- Правильность использования семантических элементов

**Примеры ошибок:**
```html
<!-- Отсутствует DOCTYPE -->
<html>
<head>
    <title>Страница</title>
</head>
</html>

<!-- Неправильный порядок -->
<body>
<head>
    <title>Страница</title>
</head>
</body>
```

### 3. **Семантическая проверка (Semantic Validation)**

Проверка правильного использования семантических элементов:

**Проверяет:**
- Правильное использование семантических тегов
- Соответствие элементов их назначению
- Правильную иерархию заголовков

**Примеры ошибок:**
```html
<!-- Неправильное использование -->
<div class="header">Шапка</div>  <!-- Должен быть <header> -->

<!-- Неправильная иерархия заголовков -->
<h1>Заголовок 1</h1>
<h3>Заголовок 3</h3>  <!-- Пропущен h2 -->
```

### 4. **Проверка доступности (Accessibility Validation)**

Проверка соответствия стандартам доступности:

**Проверяет:**
- Наличие альтернативного текста для изображений
- Правильное использование ARIA-атрибутов
- Соответствие WCAG стандартам
- Наличие меток для форм

**Примеры ошибок:**
```html
<!-- Отсутствует alt -->
<img src="photo.jpg">

<!-- Отсутствует label -->
<input type="text" name="email">

<!-- Неправильное использование ARIA -->
<div role="button" tabindex="-1">Кнопка</div>  <!-- Должен быть tabindex="0" -->
```

### 5. **Проверка атрибутов (Attribute Validation)**

Проверка правильности использования атрибутов:

**Проверяет:**
- Корректность значений атрибутов
- Обязательные атрибуты
- Неизвестные или устаревшие атрибуты
- Соответствие атрибутов элементам

**Примеры ошибок:**
```html
<!-- Неправильное значение -->
<input type="emal">  <!-- Опечатка: должно быть "email" -->

<!-- Неизвестный атрибут -->
<div align="center">  <!-- Устаревший атрибут -->

<!-- Атрибут не для этого элемента -->
<p href="#">Текст</p>  <!-- href только для <a> -->
```

### 6. **Проверка контента (Content Validation)**

Проверка содержимого элементов:

**Проверяет:**
- Допустимое содержимое элементов
- Правильную вложенность контента
- Соответствие категориям контента

**Примеры ошибок:**
```html
<!-- Блочный элемент в строчном -->
<span><div>Блок</div></span>

<!-- Неправильное содержимое -->
<p><div>Блок в параграфе</div></p>
```

### 7. **Проверка форм (Form Validation)**

Проверка правильности форм:

**Проверяет:**
- Связь `<label>` с `<input>`
- Правильность типов input
- Валидацию атрибутов форм
- Обязательные поля

**Примеры ошибок:**
```html
<!-- Label не связан с input -->
<label>Email:</label>
<input type="email" id="email">

<!-- Должно быть: -->
<label for="email">Email:</label>
<input type="email" id="email">
```

## Инструменты валидации

### 1. **W3C Markup Validator**

Официальный валидатор W3C:
- Онлайн: https://validator.w3.org/
- Проверка по URL, загрузке файла или вставке кода
- Детальный отчет об ошибках

### 2. **Валидация в браузере**

Современные браузеры показывают предупреждения в консоли разработчика.

### 3. **Расширения для редакторов**

- VS Code: HTMLHint, W3C Validation
- Sublime Text: HTML Validator
- Atom: linter-htmlhint

### 4. **Командная строка**

```bash
# Использование валидатора через npm
npm install -g html-validator
html-validator --file=index.html
```

## Процесс валидации

### 1. **Подготовка документа**

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Документ</title>
</head>
<body>
    <!-- Контент -->
</body>
</html>
```

### 2. **Запуск валидатора**

- Загрузить файл или вставить код
- Получить отчет об ошибках и предупреждениях

### 3. **Исправление ошибок**

- Исправить синтаксические ошибки
- Устранить структурные проблемы
- Улучшить семантику

### 4. **Повторная проверка**

- Проверить исправленный документ
- Убедиться, что все ошибки устранены

## Примеры валидации

### Валидный документ:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Валидная страница</title>
</head>
<body>
    <header>
        <h1>Заголовок</h1>
    </header>
    <main>
        <article>
            <h2>Статья</h2>
            <p>Текст статьи</p>
            <img src="photo.jpg" alt="Описание изображения">
        </article>
    </main>
    <footer>
        <p>Подвал</p>
    </footer>
</body>
</html>
```

### Невалидный документ:

```html
<!-- Ошибки: -->
<!-- 1. Отсутствует DOCTYPE -->
<!-- 2. Неправильная вложенность -->
<!-- 3. Отсутствует alt у изображения -->
<html>
<head>
    <title>Страница</title>
</head>
<body>
    <span>
        <div>Блок</div>
    </span>
    <img src="photo.jpg">
</body>
</html>
```

## Заключение

Валидация HTML включает проверки:
1. **Синтаксическую** — правильность синтаксиса
2. **Структурную** — правильность структуры
3. **Семантическую** — правильное использование семантики
4. **Доступности** — соответствие стандартам доступности
5. **Атрибутов** — правильность атрибутов
6. **Контента** — допустимое содержимое
7. **Форм** — правильность форм

**Преимущества валидации:**
- Обнаружение ошибок на раннем этапе
- Обеспечение совместимости
- Улучшение доступности
- Оптимизация для SEO
- Облегчение поддержки кода

Регулярная валидация HTML-документов — важная практика для создания качественных веб-страниц.

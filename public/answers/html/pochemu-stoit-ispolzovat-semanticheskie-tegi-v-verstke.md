# Почему стоит использовать семантические теги в верстке?

Семантические теги в HTML улучшают доступность, SEO, поддерживаемость кода и понимание структуры страницы как браузерами, так и разработчиками.

## Что такое семантические теги?

Семантические теги — это HTML-элементы, которые несут смысловую нагрузку и описывают назначение контента, а не только его внешний вид.

## Преимущества

### 1. **Доступность (Accessibility)**

```html
<!-- Плохо -->
<div class="header">Заголовок</div>

<!-- Хорошо -->
<header>Заголовок</header>
```

Скринридеры лучше понимают структуру страницы.

### 2. **SEO (Поисковая оптимизация)**

```html
<article>
    <h1>Заголовок статьи</h1>
    <p>Содержание...</p>
</article>
```

Поисковые системы лучше индексируют контент.

### 3. **Поддерживаемость кода**

```html
<nav>
    <ul>
        <li><a href="/">Главная</a></li>
    </ul>
</nav>
```

Код становится более читаемым и понятным.

### 4. **Автоматические функции браузера**

```html
<main>
    <article>...</article>
    <aside>...</aside>
</main>
```

Браузеры могут автоматически определять основные области страницы.

## Основные семантические теги

### Структурные:

```html
<header>Шапка</header>
<nav>Навигация</nav>
<main>Основной контент</main>
<article>Статья</article>
<section>Секция</section>
<aside>Боковая панель</aside>
<footer>Подвал</footer>
```

### Контентные:

```html
<figure>
    <img src="image.jpg" alt="Описание">
    <figcaption>Подпись</figcaption>
</figure>

<time datetime="2024-01-01">1 января 2024</time>
<mark>Выделенный текст</mark>
```

## Примеры использования

### Полная структура страницы:

```html
<!DOCTYPE html>
<html>
<head>
    <title>Страница</title>
</head>
<body>
    <header>
        <nav>Навигация</nav>
    </header>
    
    <main>
        <article>
            <h1>Заголовок</h1>
            <p>Текст статьи...</p>
        </article>
        
        <aside>Боковая панель</aside>
    </main>
    
    <footer>Подвал</footer>
</body>
</html>
```

### Статья с изображением:

```html
<article>
    <header>
        <h1>Заголовок статьи</h1>
        <time datetime="2024-01-01">1 января 2024</time>
    </header>
    
    <figure>
        <img src="image.jpg" alt="Описание">
        <figcaption>Подпись к изображению</figcaption>
    </figure>
    
    <p>Текст статьи...</p>
</article>
```

## Сравнение с `<div>`

### Плохо (не семантично):

```html
<div class="header">Заголовок</div>
<div class="content">Контент</div>
<div class="footer">Подвал</div>
```

### Хорошо (семантично):

```html
<header>Заголовок</header>
<main>Контент</main>
<footer>Подвал</footer>
```

## Заключение

**Почему использовать семантические теги:**

1. ✅ Улучшают доступность для скринридеров
2. ✅ Улучшают SEO и индексацию
3. ✅ Делают код более читаемым
4. ✅ Упрощают поддержку проекта
5. ✅ Дают браузерам больше контекста

**Основные теги:**
- `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`
- `<figure>`, `<figcaption>`, `<time>`, `<mark>`

**Рекомендации:**
- Используйте семантические теги вместо `<div>`
- Комбинируйте с правильной структурой
- Тестируйте с помощью скринридеров

Использование семантических тегов — это лучшая практика современной веб-разработки.

# Что такое семантика? Какие семантичные тэги вы знаете?

Семантика в HTML — это использование тегов, которые несут смысловое значение и описывают назначение контента, а не только его внешний вид. Семантические теги делают код более понятным для браузеров, поисковых систем, скринридеров и разработчиков.

## Что такое семантика?

Семантика — это значение и назначение элементов в HTML. Семантические теги описывают **что представляет собой контент**, а не как он выглядит. Это делает HTML более понятным и доступным.

### Разница между семантическими и несемантическими тегами:

```html
<!-- НЕсемантический подход -->
<div class="header">Шапка</div>
<div class="main">Контент</div>
<div class="footer">Подвал</div>

<!-- Семантический подход -->
<header>Шапка</header>
<main>Контент</main>
<footer>Подвал</footer>
```

## Семантичные теги HTML5

### 1. **Структурные семантические теги**

#### **`<header>`** — Шапка

```html
<header>
    <h1>Название сайта</h1>
    <nav>Навигация</nav>
</header>
```

- Шапка страницы или секции
- Может содержать логотип, навигацию, заголовки

#### **`<nav>`** — Навигация

```html
<nav>
    <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">О нас</a></li>
        <li><a href="/contact">Контакты</a></li>
    </ul>
</nav>
```

- Навигационные ссылки
- Может быть несколько на странице

#### **`<main>`** — Основной контент

```html
<main>
    <article>Статья</article>
    <section>Секция</section>
</main>
```

- Основное содержимое страницы
- Только один на странице
- Не должен быть внутри `<article>`, `<aside>`, `<footer>`, `<header>`, `<nav>`

#### **`<article>`** — Статья

```html
<article>
    <header>
        <h2>Заголовок статьи</h2>
        <time datetime="2024-01-15">15 января 2024</time>
    </header>
    <p>Содержимое статьи...</p>
</article>
```

- Независимый, самодостаточный контент
- Может быть переиспользован (RSS, блог-пост, новость)

#### **`<section>`** — Секция

```html
<section>
    <h2>Заголовок секции</h2>
    <p>Контент секции</p>
</section>
```

- Тематическая группа контента
- Обычно имеет заголовок

#### **`<aside>`** — Боковая панель

```html
<aside>
    <h3>Дополнительная информация</h3>
    <p>Боковой контент</p>
</aside>
```

- Дополнительный контент, связанный с основным
- Боковая панель, цитаты, реклама

#### **`<footer>`** — Подвал

```html
<footer>
    <p>&copy; 2024 Название сайта</p>
    <address>Контакты</address>
</footer>
```

- Подвал страницы или секции
- Может содержать авторство, ссылки, контакты

### 2. **Текстовые семантические теги**

#### **`<strong>`** — Важное выделение

```html
<p>Это <strong>важный</strong> текст</p>
```

- Семантическое выделение важности
- Визуально обычно жирный

#### **`<em>`** — Акцент

```html
<p>Это <em>акцентированный</em> текст</p>
```

- Семантическое выделение акцента
- Визуально обычно курсив

#### **`<mark>`** — Выделение

```html
<p>Это <mark>выделенный</mark> текст</p>
```

- Выделение текста для привлечения внимания
- Визуально обычно желтый фон

#### **`<cite>`** — Цитата источника

```html
<p>Как сказал <cite>Автор</cite> в своей книге</p>
```

- Цитирование источника
- Название произведения, автор

#### **`<q>`** — Короткая цитата

```html
<p>Он сказал: <q>Это цитата</q></p>
```

- Короткая встроенная цитата
- Браузер обычно добавляет кавычки

#### **`<blockquote>`** — Длинная цитата

```html
<blockquote cite="https://example.com">
    <p>Длинная цитата из источника</p>
    <footer>— Автор, <cite>Источник</cite></footer>
</blockquote>
```

- Длинная цитата
- Блочный элемент

#### **`<abbr>`** — Аббревиатура

```html
<p><abbr title="HyperText Markup Language">HTML</abbr> — язык разметки</p>
```

- Аббревиатура с расшифровкой
- При наведении показывается `title`

#### **`<dfn>`** — Определение термина

```html
<p><dfn>HTML</dfn> — язык разметки веб-страниц</p>
```

- Определение термина
- Первое упоминание термина в контексте

#### **`<code>`** — Код

```html
<p>Используйте <code>console.log()</code> для вывода</p>
```

- Фрагмент кода
- Моноширинный шрифт

#### **`<pre>`** — Предформатированный текст

```html
<pre><code>
function hello() {
    console.log('Hello');
}
</code></pre>
```

- Сохраняет форматирование (пробелы, переносы)
- Используется для кода

#### **`<kbd>`** — Клавиатурный ввод

```html
<p>Нажмите <kbd>Ctrl</kbd> + <kbd>C</kbd> для копирования</p>
```

- Клавиши или команды
- Визуально как кнопки клавиатуры

#### **`<samp>`** — Вывод программы

```html
<p>Программа вывела: <samp>Hello, World!</samp></p>
```

- Вывод программы или системы
- Моноширинный шрифт

#### **`<var>`** — Переменная

```html
<p>Переменная <var>x</var> равна 5</p>
```

- Математическая или программная переменная
- Обычно курсив

#### **`<time>`** — Время и дата

```html
<p>Опубликовано <time datetime="2024-01-15">15 января 2024</time></p>
<p>Встреча в <time datetime="2024-01-15T14:30">14:30</time></p>
```

- Машиночитаемая дата/время
- Атрибут `datetime` для формата ISO 8601

#### **`<small>`** — Мелкий текст

```html
<p>Основной текст <small>мелкий текст</small></p>
```

- Мелкий текст (юридическая информация, копирайт)
- Не для уменьшения размера, а для семантики

#### **`<sub>`** — Нижний индекс

```html
<p>H<sub>2</sub>O</p>
```

- Нижний индекс (химические формулы, математика)

#### **`<sup>`** — Верхний индекс

```html
<p>E = mc<sup>2</sup></p>
```

- Верхний индекс (степени, сноски)

#### **`<del>`** — Удаленный текст

```html
<p>Старая цена: <del>1000 руб</del></p>
```

- Удаленный или устаревший текст
- Обычно зачеркнутый

#### **`<ins>`** — Вставленный текст

```html
<p>Новая цена: <ins>800 руб</ins></p>
```

- Вставленный или новый текст
- Обычно подчеркнутый

### 3. **Медиа семантические теги**

#### **`<figure>`** — Иллюстрация

```html
<figure>
    <img src="photo.jpg" alt="Описание">
    <figcaption>Подпись к изображению</figcaption>
</figure>
```

- Группирует медиа-контент с подписью
- Может содержать изображения, видео, диаграммы

#### **`<figcaption>`** — Подпись

```html
<figure>
    <img src="photo.jpg" alt="Описание">
    <figcaption>Подпись к изображению</figcaption>
</figure>
```

- Подпись для `<figure>`
- Первый или последний дочерний элемент

### 4. **Формы и интерактивные элементы**

#### **`<label>`** — Метка формы

```html
<label for="email">Email:</label>
<input type="email" id="email" name="email">
```

- Связывает текст с элементом формы
- Улучшает доступность и UX

#### **`<output>`** — Результат вычисления

```html
<form oninput="result.value = parseInt(a.value) + parseInt(b.value)">
    <input type="number" id="a" value="5"> +
    <input type="number" id="b" value="3"> =
    <output name="result">8</output>
</form>
```

- Результат вычисления формы
- Обновляется автоматически

#### **`<meter>`** — Измеритель

```html
<meter value="0.6" min="0" max="1">60%</meter>
```

- Скалярное значение в известном диапазоне
- Прогресс, рейтинг, использование диска

#### **`<progress>`** — Прогресс

```html
<progress value="70" max="100">70%</progress>
```

- Прогресс выполнения задачи
- Загрузка, обработка

### 5. **Дополнительные семантические теги**

#### **`<address>`** — Контактная информация

```html
<address>
    Email: <a href="mailto:info@example.com">info@example.com</a>
    Адрес: ул. Примерная, 1
</address>
```

- Контактная информация
- Обычно в `<footer>`

#### **`<details>`** — Раскрывающийся блок

```html
<details>
    <summary>Нажмите для раскрытия</summary>
    <p>Скрытый контент</p>
</details>
```

- Интерактивный раскрывающийся блок
- `<summary>` — заголовок

#### **`<summary>`** — Заголовок details

```html
<details>
    <summary>Заголовок</summary>
    <p>Контент</p>
</details>
```

- Заголовок для `<details>`
- Первый дочерний элемент

#### **`<dialog>`** — Диалоговое окно

```html
<dialog open>
    <p>Содержимое диалога</p>
    <button>Закрыть</button>
</dialog>
```

- Диалоговое окно или модальное окно
- Атрибут `open` для показа

## Пример семантической структуры

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Семантическая страница</title>
</head>
<body>
    <header>
        <h1>Название сайта</h1>
        <nav>
            <ul>
                <li><a href="/">Главная</a></li>
                <li><a href="/about">О нас</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <article>
            <header>
                <h2>Заголовок статьи</h2>
                <p>Опубликовано <time datetime="2024-01-15">15 января 2024</time></p>
            </header>
            
            <section>
                <h3>Введение</h3>
                <p>Это <strong>важный</strong> текст с <em>акцентом</em>.</p>
            </section>
            
            <figure>
                <img src="photo.jpg" alt="Описание">
                <figcaption>Подпись к изображению</figcaption>
            </figure>
            
            <section>
                <h3>Основной контент</h3>
                <p>Текст с <mark>выделением</mark> и <code>кодом</code>.</p>
                <blockquote cite="https://example.com">
                    <p>Цитата из источника</p>
                </blockquote>
            </section>
        </article>
        
        <aside>
            <h3>Дополнительная информация</h3>
            <p>Боковой контент</p>
        </aside>
    </main>
    
    <footer>
        <address>
            Контакты: <a href="mailto:info@example.com">info@example.com</a>
        </address>
        <p><small>&copy; 2024 Название сайта</small></p>
    </footer>
</body>
</html>
```

## Преимущества семантических тегов

### 1. **Доступность**

- Скринридеры лучше понимают структуру
- Улучшенная навигация для людей с ограниченными возможностями

### 2. **SEO**

- Поисковые системы лучше индексируют контент
- Понимание важности и структуры контента

### 3. **Поддержка кода**

- Код более понятный и читаемый
- Легче найти нужный контент

### 4. **Автоматизация**

- Автоматическое создание оглавления
- Извлечение контента для RSS

## Заключение

**Семантика в HTML:**
- Использование тегов с смысловым значением
- Описание назначения контента, а не внешнего вида
- Улучшение доступности и SEO

**Основные семантические теги:**
- **Структурные**: `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`
- **Текстовые**: `<strong>`, `<em>`, `<mark>`, `<cite>`, `<q>`, `<blockquote>`, `<code>`, `<time>`
- **Медиа**: `<figure>`, `<figcaption>`
- **Формы**: `<label>`, `<output>`, `<meter>`, `<progress>`
- **Дополнительные**: `<address>`, `<details>`, `<summary>`, `<dialog>`

Использование семантических тегов делает HTML более понятным, доступным и оптимизированным для поисковых систем.

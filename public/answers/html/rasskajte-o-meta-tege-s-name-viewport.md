# Расскажите о meta-теге с `name="viewport"`?

Мета-тег `viewport` критически важен для адаптивного дизайна и правильного отображения веб-страниц на мобильных устройствах. Он контролирует, как браузер отображает страницу на разных размерах экрана.

## Что такое viewport?

Viewport — это видимая область веб-страницы в окне браузера. На мобильных устройствах viewport обычно уже, чем на десктопах, что требует специальной настройки.

## Мета-тег viewport

### Базовый синтаксис:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## Параметры content

### `width=device-width`

Устанавливает ширину viewport равной ширине устройства.

```html
<meta name="viewport" content="width=device-width">
```

**Без этого параметра:**
- Браузер использует ширину по умолчанию (обычно 980px)
- Страница масштабируется, чтобы поместиться на экране
- Текст и элементы выглядят маленькими

**С этим параметром:**
- Viewport равен ширине устройства
- Страница отображается в натуральном размере
- Адаптивный дизайн работает правильно

### `initial-scale=1.0`

Устанавливает начальный масштаб страницы.

```html
<meta name="viewport" content="initial-scale=1.0">
```

**Значения:**
- `1.0` — 100% (без масштабирования)
- `0.5` — 50%
- `2.0` — 200%

### `maximum-scale` и `minimum-scale`

Ограничивают масштабирование:

```html
<meta name="viewport" content="maximum-scale=2.0, minimum-scale=0.5">
```

**Не рекомендуется:**
- Ограничивает доступность
- Ухудшает UX для пользователей с нарушениями зрения

### `user-scalable`

Разрешает или запрещает масштабирование:

```html
<!-- Разрешено (по умолчанию) -->
<meta name="viewport" content="user-scalable=yes">

<!-- Запрещено (не рекомендуется) -->
<meta name="viewport" content="user-scalable=no">
```

**Не рекомендуется запрещать:**
- Нарушает доступность
- Ухудшает пользовательский опыт

## Рекомендуемая конфигурация

### Стандартная (рекомендуется):

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

### С дополнительными параметрами:

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0, user-scalable=yes">
```

## Примеры использования

### Адаптивный сайт:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Адаптивный сайт</title>
    <style>
        .container {
            width: 100%;
            max-width: 1200px;
            margin: 0 auto;
            padding: 20px;
        }
        
        @media (max-width: 768px) {
            .container {
                padding: 10px;
            }
        }
    </style>
</head>
<body>
    <div class="container">Контент</div>
</body>
</html>
```

### Без viewport (проблемы):

```html
<!-- БЕЗ viewport -->
<head>
    <meta charset="UTF-8">
    <title>Сайт</title>
</head>
```

**Результат на мобильном:**
- Страница отображается как на десктопе
- Масштабируется, чтобы поместиться
- Текст маленький и нечитаемый
- Нужно увеличивать для чтения

### С viewport (правильно):

```html
<!-- С viewport -->
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Сайт</title>
</head>
```

**Результат на мобильном:**
- Страница адаптируется под ширину устройства
- Текст читаемый
- Элементы правильного размера
- Адаптивный дизайн работает

## Влияние на CSS

### Медиа-запросы работают правильно:

```css
/* Без viewport эти запросы могут не работать */
@media (max-width: 768px) {
    .container {
        flex-direction: column;
    }
}
```

### Единицы измерения:

```css
/* vw, vh работают правильно */
.container {
    width: 100vw; /* 100% ширины viewport */
    height: 100vh; /* 100% высоты viewport */
}
```

## Проблемы без viewport

### На мобильных устройствах:

1. **Маленький текст** — страница масштабируется
2. **Горизонтальная прокрутка** — элементы не помещаются
3. **Неработающие медиа-запросы** — браузер использует десктопную ширину
4. **Плохой UX** — пользователям неудобно

## Заключение

**Мета-тег viewport:**

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

**Параметры:**
- `width=device-width` — ширина viewport = ширина устройства
- `initial-scale=1.0` — начальный масштаб 100%
- `maximum-scale` — максимальный масштаб (не рекомендуется ограничивать)
- `minimum-scale` — минимальный масштаб (не рекомендуется ограничивать)
- `user-scalable` — разрешение масштабирования (не рекомендуется запрещать)

**Рекомендации:**
- ✅ Всегда используйте viewport для адаптивных сайтов
- ✅ Используйте стандартную конфигурацию
- ✅ Не запрещайте масштабирование
- ✅ Не ограничивайте максимальный масштаб

**Критическая важность:**
- Без viewport адаптивный дизайн не работает правильно
- Страница выглядит плохо на мобильных устройствах
- Медиа-запросы могут работать некорректно

Viewport — обязательный элемент для современных адаптивных веб-сайтов.

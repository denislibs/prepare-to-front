# Что такое `<svg>` и `<canvas>`?

`<svg>` и `<canvas>` — это два HTML5 элемента для работы с графикой, но они используют разные подходы: SVG — векторная графика на основе XML, а Canvas — растровая графика через JavaScript API.

## `<svg>` — Scalable Vector Graphics

### Что это:

SVG — это векторная графика, основанная на XML. Графика описывается как набор команд и координат.

### Характеристики:

- ✅ Векторная графика (масштабируется без потери качества)
- ✅ XML-разметка (можно редактировать в коде)
- ✅ Поддержка CSS стилизации
- ✅ Поддержка анимаций через CSS/SMIL
- ✅ Доступность (текст доступен для скринридеров)
- ✅ Можно индексировать поисковыми системами

### Примеры:

```html
<svg width="200" height="200">
    <circle cx="100" cy="100" r="50" fill="blue"/>
    <rect x="50" y="50" width="100" height="100" fill="red"/>
    <text x="100" y="150" text-anchor="middle">Текст</text>
</svg>
```

## `<canvas>` — Растровая графика

### Что это:

Canvas — это растровая область для рисования через JavaScript API.

### Характеристики:

- ✅ Растровая графика (пиксели)
- ✅ Рисование через JavaScript
- ✅ Высокая производительность для сложной графики
- ✅ Поддержка 2D и 3D (WebGL)
- ❌ Не масштабируется без потери качества
- ❌ Не доступен для скринридеров напрямую

### Примеры:

```html
<canvas id="myCanvas" width="200" height="200"></canvas>

<script>
    const canvas = document.getElementById('myCanvas');
    const ctx = canvas.getContext('2d');
    
    ctx.fillStyle = 'blue';
    ctx.fillRect(10, 10, 100, 100);
    
    ctx.beginPath();
    ctx.arc(100, 100, 50, 0, 2 * Math.PI);
    ctx.fill();
</script>
```

## Сравнение

| Характеристика | SVG | Canvas |
|----------------|-----|--------|
| **Тип графики** | Векторная | Растровая |
| **Масштабирование** | Без потери качества | С потерей качества |
| **Редактирование** | Через код/CSS | Только через JS |
| **Производительность** | Хорошая для простой графики | Высокая для сложной |
| **Доступность** | Да | Нет |
| **Анимации** | CSS/SMIL | JavaScript |
| **Интерактивность** | События на элементах | Ручная обработка |

## Когда использовать что?

### Используйте SVG когда:

- ✅ Нужна векторная графика
- ✅ Нужно масштабирование без потери качества
- ✅ Простая или средняя сложность графики
- ✅ Нужна доступность
- ✅ Нужна стилизация через CSS
- ✅ Иконки, логотипы, диаграммы

### Используйте Canvas когда:

- ✅ Нужна высокая производительность
- ✅ Сложная графика с множеством элементов
- ✅ Игры и анимации
- ✅ Обработка изображений
- ✅ Графики с большим количеством данных
- ✅ Пиксельная графика

## Заключение

**SVG:**
- Векторная графика
- Масштабируется без потери качества
- Редактируется через код/CSS
- Доступен для скринридеров

**Canvas:**
- Растровая графика
- Высокая производительность
- Рисование через JavaScript
- Подходит для игр и сложной графики

Выбор зависит от задачи: SVG для векторной графики и доступности, Canvas для производительности и сложных визуализаций.

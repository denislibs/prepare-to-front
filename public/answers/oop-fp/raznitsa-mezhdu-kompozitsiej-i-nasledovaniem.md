# Разница между композицией и наследованием?

Композиция и наследование — это два разных способа организации отношений между классами. Понимание различий важно для правильного проектирования.

## Наследование

### Характеристики:

- "Является" отношением
- Класс наследует от другого класса
- Тесная связь
- Иерархия классов

```javascript
class Animal {
    speak() {
        console.log('Звук');
    }
}

class Dog extends Animal {
    // Dog "является" Animal
}
```

## Композиция

### Характеристики:

- "Имеет" отношение
- Класс содержит другой класс
- Слабая связь
- Гибкость

```javascript
class Engine {
    start() {
        console.log('Двигатель запущен');
    }
}

class Car {
    constructor() {
        this.engine = new Engine(); // Car "имеет" Engine
    }
    
    start() {
        this.engine.start();
    }
}
```

## Различия

| Характеристика | Наследование | Композиция |
|----------------|--------------|------------|
| Отношение | "Является" | "Имеет" |
| Связь | Тесная | Слабая |
| Гибкость | Низкая | Высокая |
| Изменения | Влияют на иерархию | Локальные |

## Когда использовать

### Наследование:

- ✅ Истинное отношение "является"
- ✅ Переиспользование поведения
- ✅ Полиморфизм

### Композиция:

- ✅ Отношение "имеет"
- ✅ Гибкость
- ✅ Избежание проблем наследования

## Заключение

**Наследование:**

1. ✅ "Является" отношением
2. ✅ Тесная связь
3. ❌ Меньше гибкости

**Композиция:**

1. ✅ "Имеет" отношением
2. ✅ Слабая связь
3. ✅ Больше гибкости

**Рекомендации:**

- Предпочитайте композицию наследованию
- Используйте наследование для истинных отношений
- Комбинируйте подходы при необходимости

Композиция часто предпочтительнее наследования для гибкости.

# Что такое Inversion of control?

Inversion of Control (IoC) — это принцип проектирования, при котором контроль над потоком выполнения программы инвертируется. Вместо того чтобы код вызывал библиотеки, фреймворк вызывает код.

## Что такое Inversion of Control?

IoC — это принцип, при котором контроль над выполнением передается фреймворку или контейнеру, а не управляется напрямую кодом приложения.

## Примеры

### Без IoC:

```javascript
class UserService {
    constructor() {
        this.database = new MySQLDatabase(); // Прямая зависимость
    }
}
```

### С IoC:

```javascript
class UserService {
    constructor(database) {
        this.database = database; // Зависимость инжектируется
    }
}

// Контейнер управляет созданием
const userService = container.get(UserService);
```

## Преимущества

### 1. **Слабая связность**

Компоненты не зависят от конкретных реализаций.

### 2. **Тестируемость**

Легко подменять зависимости для тестов.

### 3. **Гибкость**

Легко изменять реализации.

## Заключение

**IoC — это:**

1. ✅ Инверсия контроля
2. ✅ Управление фреймворком
3. ✅ Слабая связность

**Преимущества:**

- Слабая связность
- Тестируемость
- Гибкость

**Рекомендации:**

- Используйте для слабой связности
- Применяйте Dependency Injection
- Используйте контейнеры IoC

IoC — важный принцип для создания гибких приложений.

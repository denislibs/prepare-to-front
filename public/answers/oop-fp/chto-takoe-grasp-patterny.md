# Что такое GRASP паттерны?

GRASP (General Responsibility Assignment Software Patterns) — это набор принципов для назначения ответственности в объектно-ориентированном проектировании. GRASP помогает принимать решения о том, какой класс должен отвечать за какую функциональность.

## Что такое GRASP паттерны?

GRASP паттерны — это принципы назначения ответственности, которые помогают создавать хорошо структурированные объектно-ориентированные системы.

## Основные принципы GRASP

### 1. **Information Expert**

Назначайте ответственность классу, который имеет информацию, необходимую для выполнения задачи.

### 2. **Creator**

Класс должен создавать экземпляры классов, которые он использует.

### 3. **Controller**

Используйте класс-контроллер для обработки системных событий.

### 4. **Low Coupling**

Минимизируйте зависимости между классами.

### 5. **High Cohesion**

Классы должны иметь сильную внутреннюю связность.

### 6. **Polymorphism**

Используйте полиморфизм для обработки вариантов поведения.

### 7. **Pure Fabrication**

Создавайте классы для функциональности, не связанной с доменом.

### 8. **Indirection**

Используйте промежуточные объекты для уменьшения связности.

### 9. **Protected Variations**

Защищайте элементы от изменений через стабильные интерфейсы.

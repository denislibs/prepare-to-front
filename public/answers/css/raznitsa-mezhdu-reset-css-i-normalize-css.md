# Разница между Reset.css и Normalize.css?

Reset.css и Normalize.css — это два подхода к сбросу дефолтных стилей браузера, но они работают по-разному: Reset полностью сбрасывает стили, а Normalize нормализует их с сохранением полезных дефолтов.

## Reset.css

### Подход:

Полный сброс всех дефолтных стилей до нуля.

### Характеристики:

- ✅ Полный сброс всех стилей
- ✅ Единообразное отображение
- ✅ Начинает с чистого листа
- ❌ Удаляет полезные дефолты
- ❌ Требует больше CSS для базовых стилей

### Пример:

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

ul, ol {
    list-style: none;
}

a {
    text-decoration: none;
    color: inherit;
}
```

## Normalize.css

### Подход:

Нормализация стилей с сохранением полезных дефолтов.

### Характеристики:

- ✅ Сохраняет полезные дефолты
- ✅ Исправляет баги браузеров
- ✅ Улучшает доступность
- ✅ Меньше CSS для базовых стилей
- ✅ Более умный подход

### Пример:

```css
html {
    line-height: 1.15;
    -webkit-text-size-adjust: 100%;
}

body {
    margin: 0;
}

h1 {
    font-size: 2em;
    margin: 0.67em 0;
}
```

## Сравнение

| Характеристика | Reset.css | Normalize.css |
|----------------|-----------|---------------|
| **Подход** | Полный сброс | Нормализация |
| **Дефолты** | Удаляет все | Сохраняет полезные |
| **Браузерные баги** | Не исправляет | Исправляет |
| **Доступность** | Не улучшает | Улучшает |
| **CSS код** | Меньше | Больше |

## Когда использовать

### Reset.css:

- Когда нужен полный контроль
- Для уникального дизайна
- Когда не нужны дефолтные стили

### Normalize.css:

- Для большинства проектов (рекомендуется)
- Когда нужны полезные дефолты
- Для лучшей доступности
- Для исправления багов браузеров

## Современный подход

### Современный Reset:

```css
*, *::before, *::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
}

html {
    -webkit-text-size-adjust: 100%;
}

body {
    line-height: 1.5;
    -webkit-font-smoothing: antialiased;
}

img, picture, video, canvas, svg {
    display: block;
    max-width: 100%;
}

input, button, textarea, select {
    font: inherit;
}
```

## Заключение

**Разница:**

- **Reset.css:** Полный сброс всех стилей
- **Normalize.css:** Нормализация с сохранением полезных дефолтов

**Рекомендация:**
- Используйте Normalize.css для большинства проектов
- Используйте Reset.css для полного контроля
- Современный подход — комбинация обоих методов

Правильный выбор улучшает совместимость и уменьшает количество необходимого CSS кода.

# Разница между `GET` и `POST` запросами?

GET и POST — это два основных HTTP-метода. Понимание различий критично для правильной работы с веб-приложениями и API.

## GET

### Характеристики:

- Получение данных
- Безопасный метод
- Идемпотентный
- Данные в URL
- Кэшируемый

```javascript
GET /api/users?page=1&limit=10
```

## POST

### Характеристики:

- Отправка данных
- Небезопасный метод
- Не идемпотентный
- Данные в теле запроса
- Не кэшируемый

```javascript
POST /api/users
Body: { name: "Иван", email: "ivan@example.com" }
```

## Различия

| Характеристика | GET | POST |
|----------------|-----|------|
| Назначение | Получение | Отправка |
| Безопасность | ✅ Безопасный | ❌ Небезопасный |
| Идемпотентность | ✅ Да | ❌ Нет |
| Данные | URL | Тело запроса |
| Кэширование | ✅ Да | ❌ Нет |
| Длина данных | Ограничена | Не ограничена |

## Когда использовать

### GET:

- ✅ Получение данных
- ✅ Поиск
- ✅ Фильтрация
- ✅ Пагинация

### POST:

- ✅ Создание ресурсов
- ✅ Отправка форм
- ✅ Загрузка файлов
- ✅ Сложные данные

## Заключение

**GET:**

1. ✅ Получение данных
2. ✅ Безопасный
3. ✅ Кэшируемый

**POST:**

1. ✅ Отправка данных
2. ❌ Небезопасный
3. ❌ Не кэшируемый

**Рекомендации:**

- Используйте GET для получения
- Используйте POST для отправки
- Не используйте GET для чувствительных данных

Правильное использование методов важно для безопасности и производительности.

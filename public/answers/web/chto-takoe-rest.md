# Что такое REST?

REST (Representational State Transfer) — это архитектурный стиль для проектирования веб-сервисов. REST определяет набор принципов и ограничений для создания API, которые просты, масштабируемы и легко понимаемы.

## Что такое REST?

REST — это архитектурный стиль, который использует стандартные HTTP-методы для взаимодействия с ресурсами через URL. REST API основаны на принципах простоты, масштабируемости и единообразия.

## Основные принципы

### 1. **Ресурсы**

Все сущности представлены как ресурсы с уникальными URL.

```
GET /api/users/1
GET /api/posts/5
```

### 2. **HTTP-методы**

Использование стандартных HTTP-методов.

```
GET    - получение ресурса
POST   - создание ресурса
PUT    - обновление ресурса
DELETE - удаление ресурса
```

### 3. **Статус-коды**

Использование стандартных HTTP-статус-кодов.

```
200 OK
201 Created
404 Not Found
500 Server Error
```

### 4. **Без состояния (Stateless)**

Каждый запрос содержит всю необходимую информацию.

## Примеры

### REST API:

```javascript
// Получение пользователя
GET /api/users/1
Response: { id: 1, name: "Иван", email: "ivan@example.com" }

// Создание пользователя
POST /api/users
Body: { name: "Петр", email: "petr@example.com" }
Response: 201 Created

// Обновление пользователя
PUT /api/users/1
Body: { name: "Иван Иванов" }
Response: 200 OK

// Удаление пользователя
DELETE /api/users/1
Response: 204 No Content
```

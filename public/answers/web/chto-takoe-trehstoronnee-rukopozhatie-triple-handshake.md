# Что такое "трехстороннее рукопожатие" (Triple handshake)?

Трехстороннее рукопожатие (TCP Three-Way Handshake) — это процесс установления TCP-соединения между клиентом и сервером. Это фундаментальный механизм сетевого взаимодействия.

## Что такое трехстороннее рукопожатие?

Трехстороннее рукопожатие — это процесс установления TCP-соединения, который состоит из трех шагов обмена пакетами между клиентом и сервером.

## Процесс

### Шаг 1: SYN

Клиент отправляет SYN (synchronize) пакет серверу.

```
Клиент → Сервер: SYN
```

### Шаг 2: SYN-ACK

Сервер отвечает SYN-ACK (synchronize-acknowledge).

```
Сервер → Клиент: SYN-ACK
```

### Шаг 3: ACK

Клиент отправляет ACK (acknowledge) для подтверждения.

```
Клиент → Сервер: ACK
```

## Схема

```
Клиент                    Сервер
   |                         |
   |-------- SYN ----------->|
   |                         |
   |<----- SYN-ACK ----------|
   |                         |
   |-------- ACK ----------->|
   |                         |
   |    Соединение установлено |
```

## Зачем это нужно?

### 1. **Установка соединения**

Обе стороны подтверждают готовность к обмену данными.

### 2. **Синхронизация последовательностей**

Установка начальных номеров последовательностей.

### 3. **Надежность**

Гарантия установки соединения.

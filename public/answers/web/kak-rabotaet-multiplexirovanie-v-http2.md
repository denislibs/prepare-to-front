# Как работает мультиплексирование в `HTTP/2`?

Мультиплексирование в HTTP/2 позволяет отправлять несколько запросов одновременно в рамках одного TCP-соединения. Это одно из ключевых улучшений HTTP/2, которое значительно повышает производительность.

## Что такое мультиплексирование?

Мультиплексирование — это способность отправлять и получать несколько запросов и ответов параллельно через одно соединение.

## Как это работает

### HTTP/1.1 (без мультиплексирования):

```
Запрос 1 → Ожидание → Ответ 1
Запрос 2 → Ожидание → Ответ 2
Запрос 3 → Ожидание → Ответ 3
```

### HTTP/2 (с мультиплексированием):

```
Запрос 1, 2, 3 → Параллельная обработка → Ответы 1, 2, 3
```

## Преимущества

### 1. **Одно соединение**

Не нужно открывать несколько соединений.

### 2. **Параллельная обработка**

Несколько запросов обрабатываются одновременно.

### 3. **Лучшая производительность**

Быстрее загрузка страниц с множеством ресурсов.

### 4. **Эффективное использование соединения**

Максимальное использование пропускной способности.

## Пример

### Загрузка страницы:

```
HTTP/1.1:
- HTML (200ms)
- CSS (150ms)
- JS (100ms)
- Image 1 (80ms)
- Image 2 (90ms)
Итого: ~620ms (последовательно)

HTTP/2:
- HTML, CSS, JS, Image 1, Image 2 (параллельно)
Итого: ~200ms (параллельно)
```

**Рекомендации:**

- Используйте HTTP/2 для лучшей производительности
- Оптимизируйте количество запросов

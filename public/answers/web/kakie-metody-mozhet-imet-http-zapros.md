# Какие методы может иметь `HTTP`-запрос?

HTTP-запросы могут использовать различные методы (глаголы), которые определяют действие, которое нужно выполнить с ресурсом. Понимание методов HTTP важно для создания правильных API и веб-приложений.

## Основные методы

### 1. **GET**

Получение данных с сервера.

```javascript
GET /api/users/1
```

### 2. **POST**

Создание нового ресурса.

```javascript
POST /api/users
Body: { name: "Иван" }
```

### 3. **PUT**

Полное обновление ресурса.

```javascript
PUT /api/users/1
Body: { name: "Иван Иванов" }
```

### 4. **DELETE**

Удаление ресурса.

```javascript
DELETE /api/users/1
```

### 5. **PATCH**

Частичное обновление ресурса.

```javascript
PATCH /api/users/1
Body: { name: "Иван Иванов" }
```

### 6. **HEAD**

Получение только заголовков.

```javascript
HEAD /api/users/1
```

### 7. **OPTIONS**

Получение информации о поддерживаемых методах.

```javascript
OPTIONS /api/users
```

## Идемпотентность

### Идемпотентные методы:

- GET
- PUT
- DELETE
- HEAD
- OPTIONS

### Не идемпотентные:

- POST
- PATCH

## Безопасность

### Безопасные методы (не изменяют данные):

- GET
- HEAD
- OPTIONS

### Небезопасные методы:

- POST
- PUT
- DELETE
- PATCH

# Принципы REST-архитектуры?

REST-архитектура основана на нескольких ключевых принципах, которые обеспечивают простоту, масштабируемость и единообразие веб-сервисов. Понимание этих принципов критично для создания правильных REST API.

## Основные принципы

### 1. **Клиент-серверная архитектура**

Разделение клиента и сервера для независимого развития.

```
Клиент ←→ Сервер
```

### 2. **Без состояния (Stateless)**

Каждый запрос содержит всю необходимую информацию.

```javascript
// ❌ Неправильно (с состоянием)
// Запрос 1
GET /api/users
// Сервер запоминает пользователя

// Запрос 2
GET /api/posts
// Сервер использует запомненного пользователя

// ✅ Правильно (без состояния)
// Запрос 1
GET /api/users
Authorization: Bearer token123

// Запрос 2
GET /api/posts
Authorization: Bearer token123
```

### 3. **Кэширование**

Ответы могут быть кэшированы для улучшения производительности.

```http
HTTP/1.1 200 OK
Cache-Control: public, max-age=3600
Content-Type: application/json

{"data": "..."}
```

### 4. **Единообразный интерфейс**

Стандартные методы и форматы.

```
GET    - получение
POST   - создание
PUT    - обновление
DELETE - удаление
```

### 5. **Многоуровневая система**

Система может состоять из нескольких уровней.

```
Клиент → Прокси → Кэш → Сервер → База данных
```

### 6. **Код по требованию (опционально)**

Сервер может отправлять исполняемый код клиенту.

## Примеры

### Ресурсы:

```
GET    /api/users           - список пользователей
GET    /api/users/1         - пользователь с ID 1
POST   /api/users           - создать пользователя
PUT    /api/users/1         - обновить пользователя
DELETE /api/users/1         - удалить пользователя
```

### Статус-коды:

```javascript
// Успех
200 OK
201 Created
204 No Content

// Ошибка клиента
400 Bad Request
401 Unauthorized
404 Not Found

// Ошибка сервера
500 Internal Server Error
```

## Заключение

**Принципы REST:**

1. ✅ Клиент-серверная архитектура
2. ✅ Без состояния
3. ✅ Кэширование
4. ✅ Единообразный интерфейс
5. ✅ Многоуровневая система
6. ✅ Код по требованию

**Рекомендации:**

- Следуйте принципам REST
- Используйте стандартные HTTP-методы
- Правильно используйте статус-коды
- Реализуйте кэширование

Принципы REST обеспечивают создание простых и масштабируемых API.

# Разница между `PUT` и `POST` запросами?

PUT и POST — это два разных HTTP-метода для создания и обновления ресурсов. Понимание различий важно для правильного проектирования REST API.

## POST

### Характеристики:

- Создание нового ресурса
- Не идемпотентный
- Сервер определяет ID
- Может создавать несколько ресурсов

```javascript
POST /api/users
Body: { name: "Иван" }
Response: 201 Created
Location: /api/users/123
```

## PUT

### Характеристики:

- Полное обновление ресурса
- Идемпотентный
- Клиент определяет ID
- Заменяет весь ресурс

```javascript
PUT /api/users/123
Body: { id: 123, name: "Иван Иванов", email: "ivan@example.com" }
Response: 200 OK
```

## Различия

| Характеристика | POST | PUT |
|----------------|------|-----|
| Назначение | Создание | Обновление |
| Идемпотентность | ❌ Нет | ✅ Да |
| ID ресурса | Сервер | Клиент |
| Повторный вызов | Создает новый | Обновляет тот же |

## Когда использовать

### POST:

- ✅ Создание нового ресурса
- ✅ Неизвестен ID ресурса
- ✅ Создание подресурсов

### PUT:

- ✅ Полное обновление ресурса
- ✅ Известен ID ресурса
- ✅ Замена всего ресурса

## Заключение

**POST:**

1. ✅ Создание нового ресурса
2. ❌ Не идемпотентный
3. ✅ Сервер определяет ID

**PUT:**

1. ✅ Полное обновление ресурса
2. ✅ Идемпотентный
3. ✅ Клиент определяет ID

**Рекомендации:**

- Используйте POST для создания
- Используйте PUT для полного обновления
- Учитывайте идемпотентность

Правильное использование методов важно для качественного API.

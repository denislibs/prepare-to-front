# Разница между классовым и функциональным компонентами?

Классовые и функциональные компоненты — это два способа создания компонентов в React. Современный React предпочитает функциональные компоненты с хуками.

## Классовые компоненты

### Характеристики:

- Используют классы ES6
- Имеют состояние через `this.state`
- Используют методы жизненного цикла
- Используют `this` для доступа к пропсам и состоянию

### Пример:

```jsx
class Counter extends React.Component {
    constructor(props) {
        super(props);
        this.state = { count: 0 };
    }
    
    increment = () => {
        this.setState({ count: this.state.count + 1 });
    };
    
    render() {
        return (
            <div>
                <p>Счет: {this.state.count}</p>
                <button onClick={this.increment}>Увеличить</button>
            </div>
        );
    }
}
```

## Функциональные компоненты

### Характеристики:

- Используют функции
- Используют хуки для состояния
- Используют `useEffect` для жизненного цикла
- Проще и читабельнее

### Пример:

```jsx
function Counter() {
    const [count, setCount] = useState(0);
    
    const increment = () => {
        setCount(count + 1);
    };
    
    return (
        <div>
            <p>Счет: {count}</p>
            <button onClick={increment}>Увеличить</button>
        </div>
    );
}
```

## Сравнение

### Классовые:

- ✅ Методы жизненного цикла
- ✅ Error Boundaries
- ❌ Больше кода
- ❌ Сложнее понять

### Функциональные:

- ✅ Меньше кода
- ✅ Проще понять
- ✅ Хуки для переиспользования
- ✅ Современный подход

## Заключение

**Классовые компоненты:**

1. ✅ Классы ES6
2. ✅ this.state и методы жизненного цикла
3. ✅ Error Boundaries

**Функциональные компоненты:**

1. ✅ Функции с хуками
2. ✅ useState и useEffect
3. ✅ Современный подход

**Рекомендации:**

- Используйте функциональные компоненты
- Используйте хуки для логики
- Классовые компоненты для Error Boundaries

Функциональные компоненты — современный и предпочтительный способ создания компонентов в React.

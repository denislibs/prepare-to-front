# Разница между контролируемым и неконтролируемым компонентами в React?

Контролируемые и неконтролируемые компоненты — это два подхода к работе с формами. Контролируемые компоненты управляются React через state, а неконтролируемые — через refs и DOM.

## Контролируемые компоненты

### Характеристики:

- Состояние в React state
- Обновляется через onChange
- React контролирует значение

### Пример:

```jsx
function ControlledInput() {
    const [value, setValue] = useState('');
    
    return (
        <input
            value={value}
            onChange={(e) => setValue(e.target.value)}
        />
    );
}
```

## Неконтролируемые компоненты

### Характеристики:

- Состояние в DOM
- Доступ через refs
- React не контролирует значение

### Пример:

```jsx
function UncontrolledInput() {
    const inputRef = useRef(null);
    
    const handleSubmit = () => {
        console.log(inputRef.current.value);
    };
    
    return (
        <div>
            <input ref={inputRef} />
            <button onClick={handleSubmit}>Отправить</button>
        </div>
    );
}
```

## Заключение

**Контролируемые:**

1. ✅ State в React
2. ✅ Полный контроль
3. ✅ Валидация в реальном времени

**Неконтролируемые:**

1. ✅ State в DOM
2. ✅ Меньше кода
3. ✅ Проще для простых форм

**Рекомендации:**

- Используйте контролируемые для большинства случаев
- Используйте неконтролируемые для файловых инпутов
- Выбирайте подходящий подход

Правильный выбор зависит от требований формы.

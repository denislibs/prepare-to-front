# Разница между `React` и `ReactDOM`?

`React` и `ReactDOM` — это два отдельных пакета в экосистеме React. `React` содержит основную логику библиотеки, а `ReactDOM` отвечает за рендеринг компонентов в DOM браузера.

## Что такое React?

`React` — это основной пакет, который содержит:
- Логику компонентов
- Хуки
- API для создания элементов
- Виртуальный DOM

## Что такое ReactDOM?

`ReactDOM` — это пакет для рендеринга React-компонентов в DOM браузера.

## Различия

### React:

```jsx
import React from 'react';

// Создание элементов
const element = React.createElement('div', null, 'Hello');

// Хуки
const [state, setState] = React.useState(0);
```

### ReactDOM:

```jsx
import ReactDOM from 'react-dom/client';

// Рендеринг в DOM
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
```

## Использование

### React:

- Создание компонентов
- Хуки и состояние
- Логика приложения

### ReactDOM:

- Рендеринг в браузер
- Гидрация
- Серверный рендеринг

## Разделение

### Причины разделения:

1. **Разные платформы**
   - React — универсальный
   - ReactDOM — только для браузера
   - React Native — для мобильных

2. **Размер бандла**
   - Можно использовать React без ReactDOM
   - Меньший размер для серверного рендеринга

## Заключение

**React:**

1. ✅ Основная логика библиотеки
2. ✅ Компоненты и хуки
3. ✅ Универсальный

**ReactDOM:**

1. ✅ Рендеринг в DOM
2. ✅ Работа с браузером
3. ✅ Специфичен для веба

**Различия:**

- React — логика
- ReactDOM — рендеринг
- Разные цели и применения

**Рекомендации:**

- Используйте React для логики
- Используйте ReactDOM для рендеринга
- Понимайте разделение пакетов

Разделение React и ReactDOM позволяет использовать React на разных платформах.

# Разница между параметром и аргументом функции?

В JavaScript существует важное различие между параметрами (parameters) и аргументами (arguments) функции. Понимание этого различия критически важно для правильной работы с функциями и передачи данных.

## Что такое параметр?

Параметр — это переменная, указанная в определении функции. Это имя, которое функция ожидает получить при вызове.

### Характеристики:

- ✅ **В определении функции** — указывается при объявлении
- ✅ **Имя переменной** — это имя, которое будет использоваться внутри функции
- ✅ **Формальный** — формальное описание того, что функция принимает

### Пример:

```javascript
// a и b - это параметры
function add(a, b) {
  return a + b;
}
```

## Что такое аргумент?

Аргумент — это значение, которое передается функции при ее вызове. Это фактическое значение, которое подставляется вместо параметра.

### Характеристики:

- ✅ **При вызове функции** — передается при вызове
- ✅ **Фактическое значение** — реальное значение, которое используется
- ✅ **Соответствует параметру** — передается в соответствующий параметр

### Пример:

```javascript
// 5 и 3 - это аргументы
add(5, 3); // 5 и 3 передаются в параметры a и b
```

## Ключевые различия

### 1. **Место использования**

```javascript
// Параметры - в определении функции
function greet(name, age) { // name и age - параметры
  return `Hello, ${name}, you are ${age} years old`;
}

// Аргументы - при вызове функции
greet("John", 30); // "John" и 30 - аргументы
```

### 2. **Количество**

```javascript
function test(a, b, c) {
  // 3 параметра
}

test(1, 2); // 2 аргумента (c будет undefined)
test(1, 2, 3, 4); // 4 аргумента (4-й игнорируется)
```

### 3. **Значения по умолчанию**

```javascript
// Параметры могут иметь значения по умолчанию
function greet(name = "Guest", age = 0) {
  return `Hello, ${name}, you are ${age} years old`;
}

// Аргументы могут не передаваться
greet(); // Используются значения по умолчанию
greet("John"); // "John" и 0 (значение по умолчанию для age)
```

## Практические примеры

### Пример 1: Базовое использование

```javascript
// Параметры
function multiply(x, y) {
  return x * y;
}

// Аргументы
const result = multiply(5, 3); // 5 и 3 - аргументы
console.log(result); // 15
```

### Пример 2: Разное количество

```javascript
function sum(a, b, c) {
  // 3 параметра
  return a + (b || 0) + (c || 0);
}

sum(1, 2); // 2 аргумента, c = undefined
sum(1, 2, 3); // 3 аргумента
sum(1, 2, 3, 4); // 4 аргумента, 4-й игнорируется
```

### Пример 3: Rest parameters

```javascript
// Параметр с rest
function sum(...numbers) {
  return numbers.reduce((a, b) => a + b, 0);
}

// Множество аргументов
sum(1, 2, 3, 4, 5); // Все аргументы собираются в массив numbers
```

## Лучшие практики

### ✅ Делайте:

1. **Используйте понятные имена** — для параметров
2. **Документируйте параметры** — что они ожидают
3. **Используйте значения по умолчанию** — когда возможно
4. **Проверяйте аргументы** — на корректность

## Заключение

Разница между параметром и аргументом:

- **Параметр** — переменная в определении функции
- **Аргумент** — значение, передаваемое при вызове
- **Разные места** — параметры в определении, аргументы при вызове
- **Соответствие** — аргументы передаются в параметры

**Помните:** параметры — это имена переменных в определении функции, а аргументы — это фактические значения, передаваемые при вызове. Понимание этого различия критически важно для правильной работы с функциями.

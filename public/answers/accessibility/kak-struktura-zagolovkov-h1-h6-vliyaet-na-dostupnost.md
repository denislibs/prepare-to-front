# Как структура заголовков (H1-H6) влияет на доступность?

Структура заголовков (H1-H6) критически важна для доступности веб-страниц. Правильная иерархия заголовков помогает пользователям скринридеров навигироваться по странице и понимать структуру контента.

## Важность структуры заголовков

### Для скринридеров

Скринридеры используют заголовки для создания навигационной структуры страницы. Пользователи могут:

1. **Просмотреть список всех заголовков** — быстрая навигация
2. **Перемещаться между заголовками** — клавиатурные команды
3. **Понять иерархию контента** — структура документа

### Для всех пользователей

- Визуальная иерархия — понятная структура
- SEO — поисковые системы используют заголовки
- Стилизация — CSS для визуального оформления

## Правильная иерархия

### ✅ Хорошая структура

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <title>Главная страница</title>
</head>
<body>
  <header>
    <h1>Название сайта</h1>
    <nav>
      <h2 class="sr-only">Навигация</h2>
      <ul>
        <li><a href="/">Главная</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h1>Заголовок статьи</h1>
      <p>Введение</p>
      
      <section>
        <h2>Раздел 1</h2>
        <p>Содержимое раздела 1</p>
        
        <h3>Подраздел 1.1</h3>
        <p>Содержимое подраздела</p>
        
        <h3>Подраздел 1.2</h3>
        <p>Содержимое подраздела</p>
      </section>
      
      <section>
        <h2>Раздел 2</h2>
        <p>Содержимое раздела 2</p>
        
        <h3>Подраздел 2.1</h3>
        <p>Содержимое подраздела</p>
      </section>
    </article>
  </main>
</body>
</html>
```

### ❌ Плохая структура

```html
<!-- ❌ Пропущены уровни -->
<h1>Заголовок</h1>
<h3>Подзаголовок</h3> <!-- Пропущен h2 -->

<!-- ❌ Неправильный порядок -->
<h2>Заголовок</h2>
<h1>Главный заголовок</h1> <!-- h1 должен быть первым -->

<!-- ❌ Использование div вместо заголовков -->
<div class="heading">Заголовок</div>
```

## Правила использования

### 1. Один H1 на странице

```html
<!-- ✅ Хорошо -->
<main>
  <article>
    <h1>Заголовок статьи</h1>
    <p>Содержимое</p>
  </article>
</main>

<!-- ❌ Плохо - несколько H1 -->
<main>
  <article>
    <h1>Заголовок 1</h1>
    <h1>Заголовок 2</h1>
  </article>
</main>
```

### 2. Последовательная иерархия

```html
<!-- ✅ Хорошо - последовательная иерархия -->
<h1>Главный заголовок</h1>
  <h2>Раздел</h2>
    <h3>Подраздел</h3>
      <h4>Подподраздел</h4>
  <h2>Другой раздел</h2>
    <h3>Подраздел</h3>

<!-- ❌ Плохо - пропущены уровни -->
<h1>Главный заголовок</h1>
  <h3>Подраздел</h3> <!-- Пропущен h2 -->
```

### 3. Семантическое использование

```html
<!-- ✅ Хорошо - заголовки для структуры контента -->
<article>
  <h1>Название статьи</h1>
  <p>Введение</p>
  
  <section>
    <h2>Основная часть</h2>
    <p>Содержимое</p>
  </section>
  
  <section>
    <h2>Заключение</h2>
    <p>Содержимое</p>
  </section>
</article>

<!-- ❌ Плохо - заголовки для стилизации -->
<p>
  <h2 style="font-size: 24px; color: blue;">Важная информация</h2>
  Текст
</p>
```

## Навигация по заголовкам

### С клавиатуры

**NVDA/JAWS:**
- `H` — следующий заголовок
- `Shift+H` — предыдущий заголовок
- `1-6` — переход к заголовку определенного уровня
- `Insert+F7` — список всех заголовков

**VoiceOver:**
- `Cmd+Ctrl+H` — следующий заголовок
- `Cmd+Ctrl+Shift+H` — предыдущий заголовок
- `Cmd+Ctrl+1-6` — переход к заголовку уровня

### Программный доступ

```javascript
// Получить все заголовки
const headings = document.querySelectorAll('h1, h2, h3, h4, h5, h6');

// Создать оглавление
function createTableOfContents() {
  const toc = document.createElement('nav');
  toc.setAttribute('aria-label', 'Оглавление');
  
  const list = document.createElement('ol');
  headings.forEach((heading, index) => {
    const item = document.createElement('li');
    const link = document.createElement('a');
    link.href = `#heading-${index}`;
    link.textContent = heading.textContent;
    heading.id = `heading-${index}`;
    item.appendChild(link);
    list.appendChild(item);
  });
  
  toc.appendChild(list);
  return toc;
}
```

## Лучшие практики

### ✅ Делайте:

1. **Используйте один H1** на страницу
2. **Соблюдайте последовательность** — не пропускайте уровни
3. **Используйте семантически** — для структуры, а не стилизации
4. **Скрывайте визуально** если нужно, но оставляйте для скринридеров

### ❌ Не делайте:

1. **Не пропускайте уровни** — h1 → h3 (пропущен h2)
2. **Не используйте заголовки для стилизации** — используйте CSS классы
3. **Не создавайте несколько H1** на одной странице
4. **Не используйте div вместо заголовков**

## Заключение

Правильная структура заголовков критически важна для доступности:

- **Один H1** на страницу — главный заголовок
- **Последовательная иерархия** — не пропускайте уровни
- **Семантическое использование** — для структуры контента
- **Навигация** — скринридеры используют заголовки для навигации

**Помните:** заголовки — это не только визуальное оформление, но и важная часть структуры документа, которая помогает всем пользователям, особенно использующим вспомогательные технологии, понимать и навигировать по контенту.

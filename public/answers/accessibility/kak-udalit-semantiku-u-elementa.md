# Как удалить семантику у элемента?

Иногда необходимо использовать HTML-элемент визуально, но убрать его семантическое значение для скринридеров. Это может быть полезно, когда элемент используется не по назначению или когда семантика конфликтует с ARIA атрибутами.

## Методы удаления семантики

### 1. Атрибут `role="presentation"`

Удаляет семантическое значение элемента, но сохраняет его визуальное представление.

```html
<!-- ✅ Удалить семантику списка -->
<ul role="presentation">
  <li>Элемент 1</li>
  <li>Элемент 2</li>
</ul>
<!-- Скринридер не объявит это как список -->

<!-- ✅ Удалить семантику таблицы -->
<table role="presentation">
  <tr>
    <td>Ячейка 1</td>
    <td>Ячейка 2</td>
  </tr>
</table>
<!-- Скринридер не объявит это как таблицу -->

<!-- ✅ Удалить семантику заголовка -->
<h1 role="presentation">Заголовок</h1>
<!-- Скринридер не объявит это как заголовок уровня 1 -->
```

### 2. Атрибут `role="none"`

Эквивалент `role="presentation"`, более явно указывает на отсутствие роли.

```html
<!-- ✅ Удалить семантику -->
<div role="none">
  <p>Контент</p>
</div>

<!-- ✅ Эквивалентно -->
<div role="presentation">
  <p>Контент</p>
</div>
```

### 3. Атрибут `aria-hidden="true"`

Скрывает элемент и все его содержимое от скринридеров (более радикальный метод).

```html
<!-- ✅ Полностью скрыть от скринридеров -->
<div aria-hidden="true">
  <h2>Заголовок</h2>
  <p>Контент</p>
</div>
<!-- Скринридер полностью пропустит этот элемент -->
```

## Практические примеры

### Пример 1: Список для визуальной разметки

```html
<!-- ✅ Использование списка для визуальной разметки -->
<nav>
  <ul role="presentation" class="horizontal-menu">
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>

<!-- Альтернатива - правильная семантика -->
<nav aria-label="Основная навигация">
  <ul>
    <li><a href="/">Главная</a></li>
    <li><a href="/about">О нас</a></li>
    <li><a href="/contact">Контакты</a></li>
  </ul>
</nav>
```

### Пример 2: Таблица для визуальной разметки

```html
<!-- ✅ Таблица для визуальной разметки (не семантическая) -->
<div role="presentation">
  <table role="presentation">
    <tr>
      <td>Левая колонка</td>
      <td>Правая колонка</td>
    </tr>
  </table>
</div>

<!-- ✅ Правильная семантика для разметки -->
<div class="layout-grid">
  <aside>Левая колонка</aside>
  <main>Правая колонка</main>
</div>
```

### Пример 3: Заголовок как визуальный элемент

```html
<!-- ✅ Заголовок используется только визуально -->
<div class="card">
  <h2 role="presentation" class="card-title">
    Название карточки
  </h2>
  <p>Содержимое карточки</p>
</div>

<!-- ✅ Правильная семантика -->
<div class="card">
  <h2 class="card-title">Название карточки</h2>
  <p>Содержимое карточки</p>
</div>
```

### Пример 4: Изображение как декоративный элемент

```html
<!-- ✅ Удалить семантику изображения -->
<img 
  src="decoration.png" 
  alt=""
  role="presentation"
>

<!-- ✅ Или просто пустой alt -->
<img src="decoration.png" alt="">
```

## Когда использовать

### ✅ Используйте `role="presentation"` когда:

1. **Элемент используется не по назначению**
```html
<!-- Список для визуальной разметки -->
<ul role="presentation" class="grid">
  <li>Элемент 1</li>
  <li>Элемент 2</li>
</ul>
```

2. **Семантика конфликтует с ARIA**
```html
<!-- Таблица с кастомной ARIA ролью -->
<table role="presentation">
  <tbody role="grid">
    <tr role="row">
      <td role="gridcell">Ячейка</td>
    </tr>
  </tbody>
</table>
```

3. **Декоративные элементы**
```html
<!-- Декоративный разделитель -->
<hr role="presentation" class="decorative">
```

### ❌ НЕ используйте `role="presentation"` когда:

1. **Элемент используется по назначению**
```html
<!-- ❌ Плохо - список используется правильно -->
<ul role="presentation">
  <li>Элемент списка</li>
</ul>

<!-- ✅ Хорошо -->
<ul>
  <li>Элемент списка</li>
</ul>
```

2. **Важна семантика для доступности**
```html
<!-- ❌ Плохо - заголовок важен для структуры -->
<h1 role="presentation">Главный заголовок</h1>

<!-- ✅ Хорошо -->
<h1>Главный заголовок</h1>
```

## Разница между методами

### `role="presentation"` vs `aria-hidden="true"`

```html
<!-- role="presentation" - убирает семантику, но содержимое доступно -->
<ul role="presentation">
  <li><a href="/">Ссылка</a></li>
</ul>
<!-- Скринридер не объявит список, но прочитает ссылку -->

<!-- aria-hidden="true" - полностью скрывает элемент -->
<ul aria-hidden="true">
  <li><a href="/">Ссылка</a></li>
</ul>
<!-- Скринридер полностью пропустит этот элемент -->
```

## Вложенность и наследование

### Важно понимать:

```html
<!-- ✅ role="presentation" на родителе -->
<ul role="presentation">
  <li>Элемент 1</li>
  <li>Элемент 2</li>
</ul>
<!-- Семантика списка удалена, но элементы <li> все еще доступны -->

<!-- ✅ aria-hidden на родителе -->
<div aria-hidden="true">
  <p>Текст</p>
  <a href="/">Ссылка</a>
</div>
<!-- Все содержимое скрыто от скринридеров -->
```

## Лучшие практики

### ✅ Делайте:

1. Используйте `role="presentation"` для элементов, используемых не по назначению
2. Используйте `aria-hidden="true"` для полностью декоративных элементов
3. Предпочитайте правильную семантику вместо удаления семантики

### ❌ Не делайте:

1. Не удаляйте семантику у элементов, используемых правильно
2. Не используйте `role="presentation"` на интерактивных элементах
3. Не удаляйте семантику у важных структурных элементов (заголовки, списки навигации)

## Альтернативные подходы

### Предпочитайте правильную семантику

```html
<!-- ❌ Плохо - удаление семантики -->
<ul role="presentation" class="menu">
  <li><a href="/">Главная</a></li>
</ul>

<!-- ✅ Хорошо - правильная семантика -->
<nav aria-label="Меню">
  <ul>
    <li><a href="/">Главная</a></li>
  </ul>
</nav>
```

### Используйте семантические HTML5 элементы

```html
<!-- ❌ Плохо - div с role="presentation" -->
<div role="presentation">
  <div>Заголовок</div>
  <div>Контент</div>
</div>

<!-- ✅ Хорошо - семантические элементы -->
<article>
  <header>
    <h2>Заголовок</h2>
  </header>
  <p>Контент</p>
</article>
```

## Заключение

Удаление семантики у элементов может быть полезно в специфических случаях, но следует использовать осторожно:

- **`role="presentation"`** — убирает семантику, но сохраняет доступность содержимого
- **`role="none"`** — эквивалент `role="presentation"`
- **`aria-hidden="true"`** — полностью скрывает элемент от скринридеров

**Помните:** всегда предпочитайте правильную семантику HTML вместо удаления семантики. Удаляйте семантику только когда это действительно необходимо и когда элемент используется не по назначению.

# Уровни доступности?

Уровни доступности (Conformance Levels) в WCAG определяют степень соответствия веб-контента стандартам доступности. WCAG 2.x определяет три уровня: **A**, **AA** и **AAA**. Каждый уровень включает требования предыдущего уровня.

## Структура уровней

```
Уровень A (Минимальный)
    ↓
Уровень AA (Рекомендуемый) - включает все требования уровня A
    ↓
Уровень AAA (Максимальный) - включает все требования уровней A и AA
```

## Уровень A (Минимальный)

Базовый уровень доступности. Веб-сайт должен соответствовать всем критериям уровня A, чтобы быть доступным для людей с ограниченными возможностями.

### Основные требования уровня A:

#### 1. Текстовые альтернативы
```html
<!-- ✅ Все изображения должны иметь alt -->
<img src="logo.png" alt="Логотип компании">

<!-- ❌ Изображение без alt не соответствует уровню A -->
<img src="logo.png">
```

#### 2. Медиа-альтернативы
```html
<!-- ✅ Видео должно иметь текстовую альтернативу или субтитры -->
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="captions" src="captions.vtt" srclang="ru" default>
</video>
```

#### 3. Информация и отношения
```html
<!-- ✅ Использование заголовков для структуры -->
<h1>Главный заголовок</h1>
<h2>Подзаголовок</h2>
<h3>Подподзаголовок</h3>

<!-- ✅ Использование списков -->
<ul>
  <li>Элемент 1</li>
  <li>Элемент 2</li>
</ul>
```

#### 4. Контрастность (минимальная)
```css
/* Минимальный контраст 3:1 для крупного текста */
.large-text {
  color: #666666; /* Серый */
  background-color: #ffffff; /* Белый */
  font-size: 18pt; /* Крупный текст */
}
```

#### 5. Управление с клавиатуры
```html
<!-- ✅ Все интерактивные элементы доступны с клавиатуры -->
<button onclick="submit()" tabindex="0">Отправить</button>
<a href="/page" tabindex="0">Ссылка</a>

<!-- ❌ Элемент недоступен с клавиатуры -->
<div onclick="submit()">Отправить</div>
```

### Примеры критериев уровня A:

- **1.1.1** Нетекстовый контент: все изображения имеют текстовые альтернативы
- **2.1.1** Клавиатура: все функции доступны с клавиатуры
- **2.4.2** Заголовки страниц: каждая страница имеет описательный заголовок
- **3.3.2** Метки или инструкции: формы имеют метки

## Уровень AA (Рекомендуемый)

Средний уровень доступности. Большинство организаций и веб-сайтов стремятся достичь этого уровня, так как он обеспечивает хороший баланс между доступностью и практичностью реализации.

### Дополнительные требования уровня AA:

#### 1. Контрастность (улучшенная)
```css
/* ✅ Контраст 4.5:1 для обычного текста */
.normal-text {
  color: #333333; /* Темно-серый */
  background-color: #ffffff; /* Белый */
  /* Контраст: 12.63:1 - соответствует AA */
}

/* ✅ Контраст 3:1 для крупного текста (18pt+) */
.large-text {
  color: #666666; /* Серый */
  background-color: #ffffff; /* Белый */
  font-size: 18pt;
  /* Контраст: 5.74:1 - соответствует AA */
}

/* ❌ Низкий контраст - не соответствует AA */
.low-contrast {
  color: #cccccc; /* Светло-серый */
  background-color: #ffffff; /* Белый */
  /* Контраст: 1.6:1 - не соответствует */
}
```

#### 2. Масштабирование текста
```css
/* ✅ Текст можно увеличить до 200% без потери функциональности */
.text {
  font-size: 1rem; /* Относительные единицы */
  line-height: 1.5;
}

/* Использование viewport units для адаптивности */
.responsive-text {
  font-size: clamp(1rem, 2vw, 1.5rem);
}
```

#### 3. Множественные способы навигации
```html
<!-- ✅ Skip links -->
<a href="#main-content" class="skip-link">
  Перейти к основному контенту
</a>

<!-- ✅ Поиск -->
<form role="search">
  <input type="search" placeholder="Поиск...">
  <button type="submit">Найти</button>
</form>

<!-- ✅ Карта сайта -->
<nav aria-label="Карта сайта">
  <ul>
    <li><a href="/sitemap">Карта сайта</a></li>
  </ul>
</nav>
```

#### 4. Заголовки и метки
```html
<!-- ✅ Все формы имеют метки -->
<label for="email">Email</label>
<input type="email" id="email" name="email">

<!-- ✅ Все разделы имеют заголовки -->
<section>
  <h2>О нас</h2>
  <p>Содержимое раздела</p>
</section>
```

#### 5. Фокус видим
```css
/* ✅ Видимый фокус для всех интерактивных элементов */
button:focus,
a:focus,
input:focus {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}

/* ✅ Кастомный стиль фокуса */
.custom-button:focus {
  box-shadow: 0 0 0 3px rgba(0, 102, 204, 0.5);
}
```

### Примеры критериев уровня AA:

- **1.4.3** Контраст (минимальный): контраст 4.5:1 для обычного текста
- **1.4.4** Изменение размера текста: текст можно увеличить до 200%
- **2.4.6** Заголовки и метки: описательные заголовки и метки
- **2.4.7** Фокус видим: видимый индикатор фокуса
- **3.2.3** Последовательная навигация: последовательный порядок навигации
- **3.2.4** Идентичные функции: компоненты с одинаковой функциональностью идентифицируются одинаково

## Уровень AAA (Максимальный)

Высший уровень доступности. Достичь этого уровня для всего контента может быть сложно или непрактично, но отдельные страницы или разделы могут соответствовать уровню AAA.

### Дополнительные требования уровня AAA:

#### 1. Контрастность (высокая)
```css
/* ✅ Контраст 7:1 для обычного текста */
.aaa-text {
  color: #000000; /* Черный */
  background-color: #ffffff; /* Белый */
  /* Контраст: 21:1 - соответствует AAA */
}

/* ✅ Контраст 4.5:1 для крупного текста */
.aaa-large-text {
  color: #333333; /* Темно-серый */
  background-color: #ffffff; /* Белый */
  font-size: 18pt;
}
```

#### 2. Язык страницы
```html
<!-- ✅ Язык страницы указан -->
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Мой сайт</title>
</head>
<body>
  <p>Привет, мир!</p>
  
  <!-- Изменение языка для части контента -->
  <p>Hello, <span lang="en">world</span>!</p>
</body>
</html>
```

#### 3. Контекстная справка
```html
<!-- ✅ Подробная справка для форм -->
<form>
  <label for="password">
    Пароль
    <span class="required" aria-label="обязательное поле">*</span>
  </label>
  <input 
    type="password" 
    id="password" 
    name="password"
    aria-describedby="password-help password-requirements"
    required
  >
  
  <div id="password-help" class="help-text">
    Пароль должен содержать минимум 8 символов
  </div>
  
  <div id="password-requirements" class="requirements">
    <h3>Требования к паролю:</h3>
    <ul>
      <li>Минимум 8 символов</li>
      <li>Хотя бы одна заглавная буква</li>
      <li>Хотя бы одна цифра</li>
      <li>Хотя бы один специальный символ</li>
    </ul>
  </div>
</form>
```

#### 4. Ошибки идентификации
```html
<!-- ✅ Подробные сообщения об ошибках -->
<form>
  <label for="email">Email</label>
  <input 
    type="email" 
    id="email" 
    name="email"
    aria-invalid="true"
    aria-describedby="email-error"
    required
  >
  <span 
    id="email-error" 
    role="alert"
    aria-live="assertive"
  >
    Ошибка: Email адрес должен содержать символ @ и доменное имя. 
    Пример: user@example.com
  </span>
</form>
```

### Примеры критериев уровня AAA:

- **1.4.6** Контраст (улучшенный): контраст 7:1 для обычного текста
- **1.4.8** Визуальное представление: возможность настройки текста
- **2.4.8** Местоположение: информация о местоположении пользователя
- **3.1.3** Необычные слова: механизм для определения значения слов
- **3.3.5** Помощь: контекстная справка доступна

## Выбор уровня соответствия

### Когда использовать уровень A:
- Минимальные требования для соответствия законодательству
- Быстрая реализация базовой доступности
- Внутренние инструменты с ограниченным бюджетом

### Когда использовать уровень AA (рекомендуется):
- Публичные веб-сайты
- Коммерческие приложения
- Образовательные платформы
- Государственные сайты

### Когда использовать уровень AAA:
- Специализированные приложения для людей с ограниченными возможностями
- Критически важные сервисы (здравоохранение, финансы)
- Отдельные страницы или компоненты

## Проверка соответствия уровням

### Инструменты для проверки:

1. **WAVE** — автоматическая проверка соответствия уровням
2. **axe DevTools** — проверка критериев WCAG
3. **Lighthouse** — аудит доступности с указанием уровня
4. **Pa11y** — командная строка для проверки

### Пример отчета:

```
WCAG 2.1 Level AA Compliance Report

✅ Level A: 25/25 criteria passed (100%)
✅ Level AA: 20/25 criteria passed (80%)
⚠️  Level AAA: 5/25 criteria passed (20%)

Issues found:
- 1.4.3 (AA): Low contrast on .header-text (3.2:1, required 4.5:1)
- 2.4.7 (AA): Focus indicator not visible on .custom-button
```

## Практические рекомендации

### Начните с уровня A:
```html
<!-- Базовые требования -->
<img src="logo.png" alt="Логотип">
<h1>Заголовок</h1>
<button>Кнопка</button>
```

### Переходите к уровню AA:
```html
<!-- Улучшенная доступность -->
<img src="logo.png" alt="Логотип компании TechCorp">
<h1>Главный заголовок страницы</h1>
<button aria-label="Отправить форму">Отправить</button>
```

### Стремитесь к уровню AAA (где возможно):
```html
<!-- Максимальная доступность -->
<img src="logo.png" alt="Логотип компании TechCorp, основанной в 2020 году">
<h1>Главный заголовок страницы: О нас</h1>
<button 
  aria-label="Отправить форму регистрации"
  aria-describedby="submit-help"
>
  Отправить
</button>
<div id="submit-help" class="sr-only">
  После нажатия кнопки вы получите подтверждение на email
</div>
```

## Заключение

Уровни доступности WCAG предоставляют четкую структуру для оценки и улучшения доступности веб-контента:

- **Уровень A** — минимальные требования для базовой доступности
- **Уровень AA** — рекомендуемый уровень для большинства веб-сайтов
- **Уровень AAA** — максимальный уровень для специализированных случаев

Большинство организаций должны стремиться к уровню **AA**, который обеспечивает хороший баланс между доступностью и практичностью реализации.

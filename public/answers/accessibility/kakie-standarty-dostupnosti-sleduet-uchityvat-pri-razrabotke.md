# Какие стандарты доступности следует учитывать при разработке?

При разработке доступных веб-сайтов и приложений необходимо учитывать несколько ключевых стандартов и руководящих принципов. Эти стандарты обеспечивают единообразный подход к доступности и помогают создавать веб-контент, доступный для всех пользователей.

## Основные стандарты

### 1. **WCAG (Web Content Accessibility Guidelines)**

WCAG 2.1 и WCAG 2.2 — основные международные стандарты доступности веб-контента.

**Версии:**
- **WCAG 2.0** (2008) — базовая версия
- **WCAG 2.1** (2018) — добавлены мобильные и когнитивные требования
- **WCAG 2.2** (2023) — последняя версия с улучшениями

**Уровни соответствия:**
- **Уровень A** — минимальные требования
- **Уровень AA** — рекомендуемый уровень (большинство сайтов)
- **Уровень AAA** — максимальный уровень

**Примеры требований:**
```html
<!-- WCAG 2.1 AA: Контраст 4.5:1 -->
<p style="color: #333; background: #fff;">
  Текст с достаточным контрастом
</p>

<!-- WCAG 2.1 AA: Альтернативный текст -->
<img src="chart.png" alt="График продаж за 2023 год">

<!-- WCAG 2.1 AA: Доступность с клавиатуры -->
<button onclick="submit()" tabindex="0">
  Отправить
</button>
```

### 2. **Section 508 (США)**

Федеральный стандарт доступности для государственных веб-сайтов США.

**Требования:**
- Соответствие WCAG 2.0 Level AA
- Доступность с клавиатуры
- Альтернативный текст для изображений
- Подписи для видео

### 3. **EN 301 549 (Европа)**

Европейский стандарт доступности информационно-коммуникационных технологий.

**Требования:**
- Соответствие WCAG 2.1 Level AA
- Доступность для людей с ограниченными возможностями
- Требования к программному обеспечению

### 4. **ADA (Americans with Disabilities Act)**

Закон США о доступности, применяется к веб-сайтам коммерческих организаций.

**Требования:**
- Соответствие WCAG 2.0/2.1 Level AA
- Разумные приспособления
- Эффективная коммуникация

## Дополнительные стандарты и руководства

### 1. **ARIA (Accessible Rich Internet Applications)**

Стандарт W3C для улучшения доступности динамического контента.

```html
<!-- ✅ Правильное использование ARIA -->
<div 
  role="dialog"
  aria-labelledby="modal-title"
  aria-modal="true"
>
  <h2 id="modal-title">Диалог</h2>
</div>
```

### 2. **HTML5 Accessibility**

Стандарты доступности для HTML5 элементов.

```html
<!-- ✅ Семантические HTML5 элементы -->
<header>
  <nav aria-label="Навигация">
    <ul>
      <li><a href="/">Главная</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h1>Заголовок</h1>
    <p>Контент</p>
  </article>
</main>

<footer>
  <p>Копирайт</p>
</footer>
```

### 3. **ATAG (Authoring Tool Accessibility Guidelines)**

Руководящие принципы для инструментов создания контента.

**Применяется к:**
- CMS системы
- Редакторы кода
- Конструкторы веб-сайтов

## Практическое применение стандартов

### Чек-лист WCAG 2.1 Level AA

#### Воспринимаемость (Perceivable)

```html
<!-- ✅ 1.1.1 Нетекстовый контент -->
<img src="logo.png" alt="Логотип компании">

<!-- ✅ 1.3.1 Информация и отношения -->
<h1>Заголовок</h1>
<h2>Подзаголовок</h2>

<!-- ✅ 1.4.3 Контраст (минимум) -->
<p style="color: #333; background: #fff;">
  Контраст 4.5:1
</p>
```

#### Управляемость (Operable)

```html
<!-- ✅ 2.1.1 Клавиатура -->
<button onclick="submit()" tabindex="0">
  Отправить
</button>

<!-- ✅ 2.4.2 Заголовки страниц -->
<title>Заголовок страницы</title>

<!-- ✅ 2.4.7 Фокус видим -->
<style>
button:focus {
  outline: 2px solid #0066cc;
}
</style>
```

#### Понятность (Understandable)

```html
<!-- ✅ 3.1.1 Язык страницы -->
<html lang="ru">

<!-- ✅ 3.2.4 Идентичные функции -->
<button class="submit-btn">Отправить</button>

<!-- ✅ 3.3.2 Метки или инструкции -->
<label for="email">Email</label>
<input type="email" id="email">
```

#### Надежность (Robust)

```html
<!-- ✅ 4.1.1 Парсинг -->
<!DOCTYPE html>
<html lang="ru">
<!-- Валидный HTML -->

<!-- ✅ 4.1.2 Имя, роль, значение -->
<button 
  role="button"
  aria-label="Отправить"
  onclick="submit()"
>
  Отправить
</button>
```

## Региональные требования

### Россия

**Стандарты:**
- ГОСТ Р 52872-2019 — доступность веб-интерфейсов
- Соответствие WCAG 2.1 Level AA

### Великобритания

**Законы:**
- Equality Act 2010
- Требования к доступности публичных веб-сайтов

### Канада

**Стандарты:**
- Standard on Web Accessibility
- Соответствие WCAG 2.0 Level AA

## Процесс внедрения стандартов

### 1. **Планирование**

```markdown
1. Определить целевой уровень (обычно AA)
2. Выбрать стандарты (WCAG 2.1)
3. Составить план действий
4. Назначить ответственных
```

### 2. **Разработка**

```markdown
1. Следовать стандартам с начала разработки
2. Использовать семантический HTML
3. Добавлять ARIA атрибуты где необходимо
4. Тестировать на каждом этапе
```

### 3. **Тестирование**

```markdown
1. Автоматическое тестирование
2. Ручное тестирование
3. Тестирование с пользователями
4. Аудит доступности
```

## Заключение

Основные стандарты доступности:

- **WCAG 2.1/2.2** — основной международный стандарт
- **Section 508** — для государственных сайтов США
- **EN 301 549** — европейский стандарт
- **ADA** — закон США о доступности
- **ARIA** — для динамического контента
- **HTML5 Accessibility** — семантические элементы

**Рекомендации:**
- Стремитесь к **WCAG 2.1 Level AA** как минимум
- Тестируйте соответствие стандартам регулярно
- Документируйте соответствие
- Обновляйте стандарты при выходе новых версий

Следование этим стандартам обеспечивает доступность веб-контента для всех пользователей и помогает избежать юридических проблем.

# Для чего используется атрибут aria-roledescription?

Атрибут `aria-roledescription` используется для предоставления более понятного и локализованного описания роли элемента для пользователей скринридеров. Он позволяет заменить стандартное объявление роли на более контекстное и понятное описание.

## Основное назначение

`aria-roledescription` изменяет то, как скринридер объявляет роль элемента, делая описание более понятным и специфичным для контекста приложения.

## Как это работает

```html
<!-- ✅ Стандартное объявление -->
<div role="button" onclick="openCard()">
  Карточка продукта
</div>
<!-- Скринридер прочитает: "Карточка продукта, кнопка" -->

<!-- ✅ С aria-roledescription -->
<div 
  role="button" 
  aria-roledescription="Карточка продукта"
  onclick="openCard()"
>
  Название продукта
</div>
<!-- Скринридер прочитает: "Название продукта, Карточка продукта" -->
```

## Практические примеры

### Пример 1: Кастомные виджеты

```html
<!-- ✅ Кастомная кнопка с понятным описанием -->
<div 
  role="button"
  aria-roledescription="Карточка продукта"
  tabindex="0"
  onclick="openProduct()"
  aria-label="Открыть карточку продукта: iPhone 15"
>
  <img src="iphone.jpg" alt="">
  <h3>iPhone 15</h3>
  <p>Цена: 99,990 ₽</p>
</div>
<!-- Скринридер: "Открыть карточку продукта: iPhone 15, Карточка продукта" -->
```

### Пример 2: Локализация

```html
<!-- ✅ Локализованное описание для разных языков -->
<div 
  role="button"
  aria-roledescription="Кнопка отправки формы"
  onclick="submit()"
>
  Отправить
</div>

<!-- Английская версия -->
<div 
  role="button"
  aria-roledescription="Submit button"
  onclick="submit()"
>
  Submit
</div>
```

### Пример 3: Специфичные роли в приложении

```html
<!-- ✅ Кастомная роль для виджета -->
<div 
  role="region"
  aria-roledescription="Виджет погоды"
  aria-label="Погода в Москве"
>
  <h3>Погода</h3>
  <p>Температура: +15°C</p>
</div>
<!-- Скринридер: "Погода в Москве, Виджет погоды" -->
```

## Важные замечания

### ✅ Правильное использование

1. **Используйте для улучшения понимания**
```html
<div 
  role="button"
  aria-roledescription="Элемент навигации"
  onclick="navigate()"
>
  Главная
</div>
```

2. **Делайте описания понятными**
```html
<!-- ✅ Хорошо - понятное описание -->
<div 
  role="button"
  aria-roledescription="Кнопка добавления в корзину"
  onclick="addToCart()"
>
  В корзину
</div>
```

### ❌ Неправильное использование

1. **Не заменяйте роль полностью**
```html
<!-- ❌ Плохо - роль все еще должна быть указана -->
<div 
  aria-roledescription="Кнопка"
  onclick="submit()"
>
  Отправить
</div>

<!-- ✅ Хорошо -->
<div 
  role="button"
  aria-roledescription="Кнопка отправки"
  onclick="submit()"
>
  Отправить
</div>
```

2. **Не используйте технические термины**
```html
<!-- ❌ Плохо - непонятно для пользователя -->
<div 
  role="button"
  aria-roledescription="DOM элемент с обработчиком события click"
  onclick="submit()"
>
  Отправить
</div>

<!-- ✅ Хорошо -->
<div 
  role="button"
  aria-roledescription="Кнопка отправки формы"
  onclick="submit()"
>
  Отправить
</div>
```

## Взаимодействие с другими ARIA атрибутами

```html
<!-- ✅ Комбинация с aria-label -->
<div 
  role="button"
  aria-roledescription="Карточка"
  aria-label="Карточка продукта: iPhone 15"
  onclick="openProduct()"
>
  <h3>iPhone 15</h3>
</div>
<!-- Скринридер: "Карточка продукта: iPhone 15, Карточка" -->
```

## Лучшие практики

1. **Используйте для кастомных виджетов** — когда стандартные роли не полностью описывают элемент
2. **Делайте описания понятными** — избегайте технических терминов
3. **Локализуйте описания** — адаптируйте под язык интерфейса
4. **Не злоупотребляйте** — используйте только когда это действительно улучшает понимание

## Заключение

`aria-roledescription` — полезный инструмент для улучшения доступности кастомных виджетов и компонентов:

- **Заменяет стандартное объявление роли** на более понятное
- **Улучшает контекст** для пользователей скринридеров
- **Поддерживает локализацию** интерфейсов

**Помните:** используйте `aria-roledescription` осторожно и только когда это действительно улучшает понимание элемента пользователями скринридеров. Всегда указывайте базовую `role`, так как `aria-roledescription` только изменяет описание, а не заменяет роль.
